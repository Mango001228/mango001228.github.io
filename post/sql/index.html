SQL语法篇：

​ *通用语法规范：

​ 1.SQL语句可以单行或多行书写，以分号结尾

​ 2.SQL语句可以使用空格/缩进来增强语句的可读性

​ 3.MySQL数据库的SQL语句不区分大小写，关键字建议大写

​ 4.注释：单行：–注释内容– 或者 #注释内容

​ 多行：/注释内容/

​ *DDL数据库定义语言（数据库，表，字段）;DML数据库操作语言（增删改）;DQL数据库查询语言（查）;DCL数据库控制语言，创建数据用户、控制数据库的访问权限

**创建MySQL数据库字符集不建议设置utf8（最多三个字节），MySQL中存在特殊字符是四个字节，建议设置为uft8mb4

 *	查询所有数据库：SHOW DATABASES;
 *	查询当前数据库：SELECT DATABASE();
 *	创建：CREATE DATABASE [IF NOT EXISTS] 数据库名 [ DEFAULT CHARSET 字符集] [ COLLATE 排序规则] ；
 *	删除：DROP DATABASE [ IF EXISTS ] 数据库名；
 *	使用：USE 数据库名；

DDL—表操作—查询

  • 查询当前数据库所有表：SHOW TABLES;
  • 查询表结构：DESC 表名；
  • 查询指定表的建表语句：SHOW CREATE TABLE 表名;

DDL—表操作—创建

  • CREATE TABLE 表名（

    ​ 字段1 字段1类型 [COMMENT 字段1注释]，

    ​ 字段2 字段2类型 [COMMENT 字段2注释]，

    ​ 字段3 字段3类型 [COMMENT 字段3注释]，

    ​ ……

    ​ 字段n 字段n类型 [COMMENT 字段n注释]）

    ​ [COMMENT 表注释]；

    [……..]为可选参数

  • CREATE TABLE emp(
	id int comment '编号',
	workno varchar(10) comment '工号',
	name varchar(10) comment '姓名',
	gender char(1) comment '性别',
	age tinyint unsigned comment '年龄',
	idcard char(18) comment '身份证号',
	entrydata date comment '入职时间'
	) comment '员工表';

DDL—表操作—修改

  • 添加字段：ALTER TABLE 表名 ADD 字段名 类型（长度）[COMMENT注释] [约束]

    #为emp表增加一个新的字段“昵称”为nikename，类型为varchar(20)

    ALTER TABLE emp ADD nickname varchar(20) COMMENT '昵称';

  • 修改数据类型：ALTER TABLE 表名 MODIFY 字段名 新数据类型（长度）；

    修改字段名和字段类型：ALTER TABLE 表名 CHANGE 旧字段名 新字段名 类型（长度） [COMMENT注释] [约束]

    #将emp表的nickname字段修改为username,类型为varchar(30)

    ALTER TABLE emp CHANGE nickname username varchar(30) COMMENT '用户名';

  • 删除字段：ALTER TABLE 表名 DROP 字段名；

    #将emp表的字段username删除

    ALTER TABLE emp DROP username;

  • 修改表名：ALTER TABLE 表名 RENAME TO 新表名；

    ALTER TABLE emp RENAME TO emplyee;

  • 删除表：DROP TABLE [IF EXISTS] 表名；

    删除指定表，并重新创建该表：TRUNCATE TABLE 表名； —————数据删除，保留表结构

  • 关键字：select

DQL-基本查询

  • 查询多个字段：SELECT 字段1，字段2，字段3…FROM 表名；

      				  SELECT*FROM 表名；

    ​

    select name,workno,age from emplyee;

select *from emplyee;
//尽量不使用通配符*，影响效率且不直观;

  • 设置别名：SELECT 字段1 [AS别名1]，字段2 [AS别名1] … FROM 表名；

    select workaddress as '工作地址' from emplyee;
select workaddress '工作地址' from emplyee;
//as可以省略

  • 去除重复记录：SELECT DISTINCT 字段列表 FROM 表名；

    select distinct workaddress '工作地址' from emplyee;

DQL-聚合函数

  • count ——统计数量
  • max ——最大值
  • min ——最小值
  • avg ——平均值
  • sum ——求和
  • null值不参与计算

语句：SELECT 聚合函数 (字段列表) FROM 表名；

1.统计该企业员工的数量
select count(*) from emplyee;

2.统计该企业员工的平均年龄
select avg(age) from emplyee;

3.统计该企业的最大年龄
select max(age) from emplyee;

4.统计该企业员工的最小年龄
select min(age) from emplyee;

5.统计西安地区员工的年龄之和
select  sum(age) from emplyee where workaddress = '西安';

DQL-分组查询

  • 语法：SELECT 字段列表 FROM 表名 [WHERE 条件] GROUP BY 分组字段名 [HAVING 分组后过滤条件]

  • where与having的区别：1.执行时机不同：where是分组前进行过滤；having是分组后过滤

      								  2.判断条件不同：where不能对聚合函数进行判断，而having可以

    据性别分组，统计男性员工和女性员工的数量
select gender , count(gender) from emplyee group by gender;

2.根据性别分组，统计男性员工和女性员工的平均年龄
select gender,avg(age) from emplyee group by gender;

3.查询年龄小于45岁的员工，并根据工作地址分组，获取员工数量大于等于3的工作地址
select workaddress,count(workaddress) from emplyee where age < 45 group by  workaddress having count(workaddress) >= 3;

DQL-排序查询

  • 语法：SELECT 字段列表 FROM 表名 ORDER BY 字段1 排序方式1，字段2 排序方式2；

  • ASC:升序

    DESC:降序

1.根据年龄对公司的员工进行升序排序
select name,age from emplyee order by age asc ;

2.根据入职时间，对员工进行降序排序
select name,entrydata from emplyee order by entrydata desc;

3.根据年龄对公司的员工进行升序排序，年龄相同，再按照入职时间进行降序排序
select name,age,entrydata from emplyee order by age ,entrydata desc;
